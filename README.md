# w2f — Way To Freedom

ربات تلگرام فروش و مدیریت سرویس روی پنل **3x-ui**.
محصول اصلی **کانفیگ رزیدنتال** است: هر سرویس یک اینباند **VLESS xHTTP TLS** با
پورت تصادفی می‌سازد و آن را از طریق یک **اوتباند اختصاصی SmartProxy** (با
روتینگ‌رول مخصوص همان کاربر) خارج می‌کند؛ پس هر کاربر IP و لوکیشن مستقل دارد و
می‌تواند هر زمان IP، کشور، استان، شهر و زمان تعویض IP را تغییر دهد. بخش
**V2Ray عادی** نیز به‌عنوان محصول دوم اسکلت‌بندی شده (پنل آن بعداً اضافه می‌شود).

## نصب با یک دستور

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/W2F-Sa/resibot/main/install.sh)
```

نصب‌کننده همه‌ی مقادیر لازم (توکن ربات، ادمین، پنل، SmartProxy، قیمت‌ها و کلیدهای
NowPayments) را به‌صورت تعاملی می‌پرسد، سرویس `systemd` می‌سازد و ربات را اجرا می‌کند.

## آپدیت بدون از دست رفتن دیتابیس

```bash
bash /opt/resibot/update.sh
```

دیتابیس در `data/` و تنظیمات در `.env` حفظ می‌شوند و قبل از هر آپدیت پشتیبان گرفته
می‌شود. مهاجرت اسکیمای دیتابیس additive و با `PRAGMA user_version` است (نسخه ۲:
کاربران/نقش‌ها، کیف پول، درخواست همکاری، پرداخت‌ها — بدون از دست رفتن داده‌ی قبلی).

## امکانات

- **باز برای همه**: هر کاربری می‌تواند سرویس بخرد، کیف پول شارژ کند و سرویس‌هایش را مدیریت کند.
- **نقش‌ها**: ادمین، همکار رزیدنتال (پس‌پرداخت، تسویه با ادمین)، همکار V2Ray
  (پیش‌پرداخت با کیف پول)، کاربر عادی (پرداخت از کیف پول).
- **درخواست همکاری**: کاربر با توضیح درخواست می‌دهد؛ ادمین تأیید/رد می‌کند. همکار
  V2Ray باید حداقل موجودی داشته باشد (قابل تنظیم توسط ادمین).
- **قیمت‌گذاری چندلایه** و قابل ویرایش در ربات: رزیدنتال عادی/همکار و V2Ray عادی/همکار.
- **کیف پول + شارژ با NowPayments** (وب‌هوک IPN با تأیید امضای HMAC؛ شارژ idempotent).
- **خرید** با انتخاب کشور (با جستجو)، استان و شهر **از لیست** (بدون تایپ دستی) و زمان تعویض IP.
- **تمدید/افزایش حجم**: افزودن حجم (با حداقل قابل تنظیم) + افزودن ۳۰ روز به اعتبار.
- **تغییر IP / کشور / استان / شهر / زمان تعویض IP** بعد از خرید + **تست اتصال (پینگ)**.
- لینک **Subscription** + لینک مستقیم دقیق (با `allowInsecure`).
- گواهی پنل برای TLS («Set as panel») تا اتصال بدون خطای SSL باشد.
- امنیت: کوئری‌های پارامتری (ضد SQL Injection)، تأیید امضای IPN، اعتبارسنجی ورودی‌ها و escape متن کاربر.

## NowPayments

برای فعال شدن شارژ کیف پول این مقادیر در `.env` لازم است: `NOWPAYMENTS_API_KEY`،
`NOWPAYMENTS_IPN_SECRET` و `PUBLIC_BASE_URL`. آدرس callback که باید در داشبورد
NowPayments ثبت شود:

```
<PUBLIC_BASE_URL>/nowpayments/ipn
```

ربات یک سرور سبک روی `IPN_PORT` (پیش‌فرض 8090) بالا می‌آورد تا IPN را دریافت کند.
بهتر است این پورت پشت HTTPS (ریورس‌پروکسی) قرار گیرد.

## مدیریت سرویس

```bash
journalctl -u resibot -f        # لاگ زنده
systemctl restart resibot       # ری‌استارت
```

## ساختار

```
bot/
  config.py       env + اعتبارسنجی
  database.py     SQLite + مهاجرت (settings/users/configs/payments/requests)
  panel.py        کلاینت 3x-ui (لاگین CSRF + توکن)
  proxy.py        ساخت username اسمارت‌پروکسی + لوکیشن
  locations.py    دیتاست کشور/استان/شهر
  countries.py    لیست کامل کشورها + جستجو
  xray_config.py  اوتباند + روتینگ‌رول هر کاربر
  inbound.py      اینباند VLESS xhttp tls + لینک ساب/مستقیم
  nowpayments.py  کلاینت درگاه + تأیید امضای IPN
  ipn.py          وب‌هوک IPN (aiohttp)
  service.py      منطق اصلی (خرید/تمدید/کیف پول/قیمت/پنل)
  middlewares.py  ثبت کاربر + تزریق نقش
  handlers/       common / admin / order / configs / wallet
  main.py         نقطه‌ی ورود (polling + IPN)
install.sh        نصب تعاملی یک‌دستوری
update.sh         آپدیت با حفظ دیتابیس
```
